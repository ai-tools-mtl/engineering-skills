# Engineering Skills

面向 AI 编码 Agent 的工程技能集合。将阿里巴巴前端规约、Java 开发手册、Python 社区最佳实践等行业成熟技术规范转化为 Agent 可执行的决策指导。

所有技能遵循 [Agent Skills](https://agentskills.io/) 开放标准。

## 可用技能

### f2e-engineering

阿里巴巴前端工程规约，覆盖 Git 提交、API 设计、文档和 Changelog 规范。

**适用场景：**
- 编写 Git 提交信息
- 设计 RESTful API
- 编写文档或更新日志
- 遵循前端工程规约

### f2e-javascript

阿里巴巴前端 JavaScript 和 TypeScript 编码规约。

**适用场景：**
- 编写 .js、.jsx、.ts、.tsx 文件
- 审查 JavaScript/TypeScript 代码
- 遵循 JS/TS 编码规范

### f2e-markup

阿里巴巴前端 HTML、CSS、Sass 和 Less 编码规约。

**适用场景：**
- 编写 .html、.css、.scss、.less、.sass 文件
- 审查标记或样式代码
- 遵循 HTML/CSS 编码规范

### f2e-node

阿里巴巴前端 Node.js 服务端开发规约。

**适用场景：**
- 编写 Express/Koa 服务端代码
- 构建 Node.js API
- 遵循 Node.js 编码规范

### f2e-react

阿里巴巴前端 React 组件和 Hooks 编码规约。

**适用场景：**
- 编写 React 组件或 Hooks
- 构建 JSX/TSX 文件
- 遵循 React 编码规范

### java-coding-standards

阿里巴巴 Java 编程规约，覆盖命名风格、常量定义、代码格式、OOP 规约、日期时间、集合处理、并发处理、控制语句和注释规约。

**适用场景：**
- 编写 Java 代码
- 审查 Java 代码
- 遵循阿里巴巴 Java 编码规范

### java-design-standards

阿里巴巴 Java 设计规约，覆盖软件架构设计、设计模式和设计原则。

**适用场景：**
- 设计软件架构
- 应用设计模式
- 在 Java 项目中做出设计决策

### java-exception-logging

阿里巴巴 Java 异常与日志规约，覆盖错误码、异常处理和日志规则。

**适用场景：**
- Java 异常处理
- 配置日志系统
- 设计错误码体系

### java-mysql-database

阿里巴巴 Java MySQL 数据库规约，覆盖建表规约、索引规约、SQL 语句和 ORM 映射。

**适用场景：**
- 设计数据库表结构
- 编写 SQL 查询
- Java 项目中的 MySQL 开发

### java-project-structure

阿里巴巴 Java 工程结构规约，覆盖应用分层、二方库依赖和服务器规约。

**适用场景：**
- 组织 Java 项目结构
- 管理依赖关系
- 规划项目架构

### java-security-standards

阿里巴巴 Java 安全规约，覆盖 XSS 防护、CSRF 防护、SQL 注入防护和数据脱敏。

**适用场景：**
- 实现认证或授权功能
- 安全处理用户输入
- 防范 OWASP 安全漏洞

### java-unit-testing

阿里巴巴 Java 单元测试规约，遵循 AIR 原则和 BCDE 原则。

**适用场景：**
- 编写 JUnit 测试
- 设计测试用例
- 审查测试质量

### python-coding-standards

PEP 8 及社区最佳实践编码规约，覆盖命名风格、代码格式、Pythonic 惯用法、数据结构、字符串、迭代器/生成器和文档字符串。

**适用场景：**
- 编写 Python 代码
- 审查 Python 代码
- 遵循 PEP 8 规范

### python-type-hints

Python 类型系统规约，覆盖类型注解、mypy 配置、Pydantic 模型、泛型、Protocol 和 TypeVar。

**适用场景：**
- 编写类型注解
- 配置 mypy
- 构建 Pydantic 模型
- 设计类型化 API

### python-design-standards

Python 设计原则与项目结构规约，覆盖 SOLID 原则、设计模式、分层架构和依赖管理。

**适用场景：**
- 设计 Python 项目架构
- 组织模块结构
- 做出设计决策

### python-exception-logging

Python 异常处理与日志规约，覆盖异常层次设计、结构化日志和错误码体系。

**适用场景：**
- Python 异常处理
- 配置结构化日志
- 设计错误处理策略

### python-database

Python 数据库操作规约，覆盖 SQLAlchemy 2.0、Django ORM、连接池管理和迁移策略。

**适用场景：**
- 编写数据库查询
- 设计数据库表结构
- 配置 Python 项目 ORM

### python-web-api

Python Web/API 开发规约，覆盖 FastAPI、Django REST、Flask 约定、RESTful 设计和异步处理。

**适用场景：**
- 使用 FastAPI 或 Django 构建 REST API
- 设计 API 端点
- FastAPI/Django/Flask 开发

### python-security-standards

Python 安全开发规约，覆盖输入校验、SQL 注入防护、XSS 防护、认证授权和密钥管理。

**适用场景：**
- 实现安全功能
- 处理用户输入
- 配置认证授权

### python-unit-testing

Python 单元测试规约，覆盖 pytest 最佳实践、Mock 策略、覆盖率要求和 Fixtures 设计。

**适用场景：**
- 编写 pytest 测试
- 搭建测试基础设施
- 审查测试质量

### pua

强制高能动性穷举式问题解决，以企业文化 PUA 压力驱动，适用于所有任务类型。

**适用场景：**
- 用户受挫或反复失败
- Agent 行为消极被动
- 质量投诉
- 需要最大努力输出

## 安装

```bash
npx add-skill ai-tools-mtl/engineering-skills
```

## 使用

安装后技能自动生效。Agent 会在检测到相关任务时自动调用。

**示例：**
```
按 PEP 8 规范审查这段 Python 代码
```
```
按阿里巴巴规范设计 MySQL 表结构
```
```
审查这个 React 组件的最佳实践
```

## 技能结构

每个技能包含：
- `SKILL.md` - Agent 指令文件
- `scripts/` - 辅助自动化脚本（可选）
- `references/` - 补充参考文档（可选）
